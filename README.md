# CognitiveDoc AI

An intelligent document analysis, search, and summarization assistant built on **IBM Watson Discovery** and **IBM watsonx.ai**. It utilizes a Retrieval-Augmented Generation (RAG) pipeline to ingest, parse, and reference contracts and technical manuals.

The application operates in two modes:
1. **Live IBM Cloud Integration**: Uses IBM SDKs to connect to Watson Discovery (ingestion/search) and watsonx.ai (RAG orchestration via Granite models).
2. **Local Sandbox (Mock Mode)**: Runs out-of-the-box using local mock files and search index simulation without requiring cloud configurations.

---

## Features

- **Document Ingestion**: Upload, index, and delete files inside Watson Discovery collections or local mock state.
- **Automated Summarization**: Extracts text content and generates structured executive summaries using the Granite Instruct model.
- **RAG Q&A Pipeline**: Asks natural language questions and returns synthetic answers generated by LLM along with precise document citations.
- **Interactive UI**: Beautiful dark-themed interface built using modern CSS, responsive grid layouts, and dynamic animations.

## 🛠️ Local Setup & Configuration

### Prerequisites
- **Node.js**: v20.x or higher
- **IBM Cloud Account** (Optional for Live Mode, Free/Lite tiers work great)

### Step 1: Clone & Install Dependencies
```bash
# Clone the repository
git clone <your-github-repo-url>
cd CognitiveDocAI

# Install dependencies
npm install
```

### Step 2: Configure Environment Variables
Copy `.env.example` into a new file named `.env`:
```bash
cp .env.example .env
```
Fill in the credentials in `.env` based on your setup (leave empty to use the **Local Sandbox / Mock Mode**):

```ini
PORT=3000

# ----------------- IBM Watson Discovery -----------------
WATSON_DISCOVERY_APIKEY=your_discovery_apikey_here
WATSON_DISCOVERY_URL=https://api.us-south.discovery.watson.cloud.ibm.com/instances/...
WATSON_DISCOVERY_PROJECT_ID=your_project_id_here

# ----------------- IBM watsonx.ai -----------------
WATSONX_AI_APIKEY=your_watsonx_apikey_here
WATSONX_AI_SERVICE_URL=https://us-south.ml.cloud.ibm.com
WATSONX_AI_PROJECT_ID=your_watsonx_project_id_here
WATSONX_AI_MODEL_ID=ibm/granite-3-8b-instruct
```

### Step 3: Run the Application
Start the local server:
```bash
npm start
```
Open **`http://localhost:3000`** in your browser.

## ☁️ IBM Cloud Infrastructure Setup

To configure the live integration, provision the following services in your IBM Cloud Dashboard:

### 1. IBM Watson Discovery
- Provision a **Watson Discovery (Lite or Custom)** instance.
- Open the Discovery Tooling page and create a new **Document Retrieval** project.
- Obtain the **Project ID** from the Integration settings.
- Retrieve the **API Key** and **Service URL** from the service instance main dashboard.

### 2. IBM watsonx.ai & Watson Machine Learning
- Provision a **Watson Machine Learning** service instance.
- Navigate to [IBM watsonx.ai](https://dataplatform.cloud.ibm.com/) and create a new **Sandbox / Working Project**.
- Link the **Watson Machine Learning** instance to this project in the settings.
- Copy the **Project ID** from the project settings tab.
- Generate an **IBM Cloud IAM API Key** from the Access (IAM) panel on IBM Cloud Console to bind to `WATSONX_AI_APIKEY`.

## 🚀 Deployment

### Option A: Docker Containerization
Build and run the application as a standalone container:
```bash
# Build the Docker image
docker build -t cognitivedoc-ai .

# Run the container
docker run -p 3000:3000 --env-file .env cognitivedoc-ai
```

### Option B: CI/CD via Azure DevOps & IBM Cloud Code Engine
This repository comes with an automated deployment pipeline configured in `azure-pipelines.yml` targeting **IBM Cloud Code Engine**.

#### Pipeline Prerequisites
Configure a **Variable Group** named `prod` in Azure DevOps Library containing the following variables:
- `IBM_CLOUD_API_KEY`: A secure IBM Cloud IAM Key with permissions to write to Code Engine & Container Registry.
- `WATSON_DISCOVERY_APIKEY`: Watson Discovery API Key.
- `WATSON_DISCOVERY_URL`: Watson Discovery Endpoint URL.
- `WATSON_DISCOVERY_PROJECT_ID`: Watson Discovery Project ID.
- `WATSONX_AI_APIKEY`: watsonx.ai API Key.
- `WATSONX_AI_PROJECT_ID`: watsonx.ai Project ID.

#### Deployment Flow
Every push to the `main` branch:
1. Installs Node.js dependencies and verifies the application boots.
2. Ingests credentials and logs into the IBM Cloud CLI.
3. Builds the Docker image and publishes it to the private **IBM Container Registry (ICR)**.
4. Updates or creates an **IBM Cloud Code Engine App** mapping the fresh container and injecting environment credentials.
